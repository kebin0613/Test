<HTTP 통신>

-사전적 정의 
  (Hyper Text Transfer Protocol)
  Hyper Text 를 전송하기 위한(데이터를 주고 받기위한) 프로토콜
  Hyper text는 웹 문서를 구성하고 있는 언어인 HTML을 의미

-데이터는 오디오 비디오 이미지 텍스트 등 데이터의 종류가 다양

-HTTP 통신은 요청(Request),응답(Responce)으로 구성

-OSI7계층 중 4계층
 4계층에서 정보를 만들어 전달
 3계층에서 통신노드를 연결
 2계층에서 노드간 전송과 라우팅
 1계층에서 전기적 신호로 변환하여 실제로 전달
  (받는쪽에선 반대로)

-HTTP Method(메소드)
 GET - URL이 가진 정보를 검색하기 위해 서버 측에 요청하는 형태
 POST - POST 방식은 요청 URL에 폼 입력을 처리하기위해 구성한 서버측 스크립트
 HEAD - HEAD 요청 방식은 GET과 유사한 방식이나 웹서버에서 헤더 정보 이외에는
        어떤 데이터도 보내지 않는다.
 OPTION- 해당 메소드를 통해 시스템에서 지원되는 메소드 종류를 확인할 수 있다. 
 PUT - POST와 유사한 전송 구조를 가지기 때문에 헤더 이외에 메세지(데이터)가 
       함께 전송된다. 원격지 서버에 지정한 콘텐츠를 지정하기 위해 사용되며
       홈페이지 변조에 많이 악용되고있다.
 DELETE - 원격지 웹서버에 파일을 삭제하기 위해 사용하며 PUT과는 반대 개념의 메소드
 TRACE - 원격지 서버에 루프백 메세지를 호출하기 위해 사용된다.
 CONNECT - 웹 서버에 프락시 기능을 요청할 때 사용된다.

 (루프백 IP주소 = LOCALHOST 127.0.0.1 을 사용)
                  네트워크 상에서 자기 자신을 나타내는 인터페이스,주소
 EX) LOCALHOST 8000 --> 포트번호가 8000번

 (javascript object notation)
 1.name-value 쌍의 포멧으로 구조화된 정보를 인코딩 하는 규격

-리소스 정보중
 General : 요청 URL 정보와 메소드, 상태코드 확인
 Response Headers : 응답헤더, 응답 온 패킷의 헤더를 확인할 수 있다.(서버의 종류, 연결상태)
 Request Headers : 요청헤더, 요청을 보낸 패킷의 헤더를 확인할 수 있다.(보낸 클라이언트의 종류)
